# 颜色设计系统

## 概述

本设计系统为南科大足球平台微信小程序定义了一套完整的橙色系配色方案，确保项目在视觉上保持一致、可维护且符合现代设计原则。所有颜色元素统一使用橙色系，不再使用深绿色 (`#003f43`) 和青绿色 (`#2bb7b3`)。

### 设计原则

1. **一致性**：所有组件、页面和元素使用统一的颜色调色板
2. **层次性**：通过颜色的深浅、透明度建立清晰的视觉层次
3. **可用性**：确保颜色对比度符合可访问性标准（WCAG 2.1 AA级）
4. **可维护性**：通过集中管理的颜色常量简化更新和维护
5. **现代化**：使用渐变、阴影等现代设计元素增强视觉吸引力

### 卡片组件

1. **标题文字**：主橙色 (`#ed6c00`)

6. **次要文本**：中灰 (`#cccccc`)

## 实施指南

### JavaScript 颜色常量

颜色常量定义在 `wechat-miniprogram/utils/colors.js` 文件中：

```javascript
// 导入颜色常量
import { Colors, Shadows, Gradients } from '../../utils/colors.js';

// 使用示例
Page({
  data: {
    primaryColor: Colors.primary,           // #ed6c00
    primaryLightColor: Colors.primaryLight, // #ff8c32
    cardShadow: Shadows.card,               // 0 4rpx 24rpx rgba(237, 108, 0, 0.1)
    primaryGradient: Gradients.primary,     // linear-gradient(90deg, #ed6c00 0%, #ff8c32 100%)
  }
});
```

### WXSS 混入类

混入类定义在 `wechat-miniprogram/styles/mixins.wxss` 文件中：

```css
/* 在组件的 WXSS 文件中导入混入 */
@import '../../styles/mixins.wxss';

/* 使用示例 */
.card-title {
  .primary-color();
  font-weight: bold;
}

.card-container {
  .primary-border();
  .gradient-primary();
  .shadow-card();
  background: var(--gradient-bg);
}

.primary-button {
  .primary-bg();
  .primary-light-color();
  border-radius: 24rpx;
}
```

## 最佳实践

### 可访问性

1. **文本对比度**：确保文本与背景的对比度至少达到 4.5:1（AA级）
2. **交互状态**：为所有交互元素提供明显的悬停、聚焦、激活状态
3. **颜色不唯一**：不使用颜色作为传达信息的唯一方式

### 性能优化

1. **减少重复**：使用颜色常量和混入类避免重复的颜色值
2. **渐变更用**：合理使用渐变，避免过度使用影响性能
3. **阴影优化**：使用合适的阴影参数，避免过度模糊影响渲染性能

### 维护性

1. **单一来源**：所有颜色值仅定义在颜色常量文件中
2. **命名一致性**：使用统一的命名约定（如 `primary`, `primaryLight`, `primaryLighter`）
3. **文档同步**：颜色变更时同步更新设计系统文档

## 常见问题

### Q: 微信小程序支持 CSS 变量吗？
A: 微信小程序的 WXSS 支持 CSS 自定义属性（CSS 变量）。我们使用双轨制：JavaScript 常量为主方案，CSS 变量为备选方案。

### Q: 如何添加新的颜色变体？
A: 在 `colors.js` 文件中添加新的颜色常量，在 `mixins.wxss` 中添加对应的混入类，并更新设计系统文档。

### Q: 现有组件如何迁移到颜色系统？
A: 在组件现代化改造过程中进行迁移：1）更新 WXML 内联样式使用颜色常量，2）更新 WXSS 使用混入类，3）测试视觉一致性。

### Q: 如何确保设计稿与实现一致？
A: 设计团队在设计稿中使用相同的颜色常量名称，开发团队在实现时引用对应的颜色常量。
